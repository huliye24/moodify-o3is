## 5.1 我们不做什么样的产品

在说出"Moodify 做什么"之前，先说清楚"Moodify 不做什么"。

### 我们不是"音乐播放器"

```
✗ 我们不是 Spotify
  ——不追求歌单数量
  ——不追求推荐算法
  ——不追求"更多更好"

✗ 我们不是"白噪音App"
  ——不提供"专注"、"助眠"、"学习"等功能标签
  ——我们不是工具，我们是陪伴

✗ 我们不是"情绪调节App"
  ——不承诺"用音乐治愈焦虑"
  ——不承诺"3分钟快速入睡"
  ——不承诺任何"效果"
```

### 为什么我们拒绝这些定位？

```
因为每一种定位，
都暗示了一种"你应该更好"的目的。

音乐播放器暗示：找歌听
白噪音App暗示：提高专注/睡眠质量
情绪调节App暗示：消除负面情绪

但 Moodify 不想让人"更好"。

Moodify 想让人"在现在的状态里，感到安心"。

这是本质的区别。
```

## 5.2 我们做什么样的产品

Moodify 是一种**"情绪容器"**。

```
┌─────────────────────────────────────────┐
│                                         │
│   情绪容器                               │
│                                         │
│   情绪 → 进入容器 → 被容纳 → 感到安心     │
│                                         │
│   情绪不被消除                           │
│   情绪只是被看见、被陪伴                 │
│                                         │
└─────────────────────────────────────────┘
```

具体来说，Moodify 的产品包含以下形态：

---

## 5.3 产品形态一：Moodify 情绪专辑

### 核心概念

```
每个情绪状态 = 一张专辑
每张专辑 = 8-12 首演化循环音乐

情绪专辑 ≠ 歌单
情绪专辑 = 一段完整的情绪旅程
```

### 情绪专辑的命名

```
不叫"助眠音乐"
不叫"专注音乐"
不叫"放松音乐"

我们叫它：

蜷缩
迷茫
觉醒
舒展

或者其他更诗意的名字。
```

### 示例：一张"蜷缩"专辑

```
专辑名称：蜷缩

封面意象：
深夜的窗台，一盏小灯，
有一个人坐在角落，
把膝盖抱在胸前。

音乐结构：
Track 1-3：蜷缩（低频包裹，循环不变）
Track 4：转折点（一个单音钢琴，像敲门）
Track 5-8：迷茫 → 觉醒（频率缓慢打开）

时长：约 45-60 分钟

使用场景：
- 失眠的夜晚
- 情绪崩溃后
- 什么都不想做的下午
- 蜷在被子里的时候
```

### 情绪专辑的封套设计原则

```
✓ 色彩：低饱和度，像褪色的照片
✓ 意象：具体但不明确的场景
✓ 留白：大量留白，让情绪有呼吸空间
✓ 文字：少而克制，不解释，只呈现
```

## 5.4 产品形态二：Moodify 时刻

### 核心概念

```
时刻 ≠ 音乐
时刻 = 音乐 + 场景 + 意图

但意图不是"解决问题"，
意图是"提供陪伴"。
```

### 时刻的分类

```
类别一：日常时刻
━━━━━━━━━━━━━━━━━━━━━━━━
- 晨间清醒（不是起床激励）
- 午后放空（不是下午茶背景）
- 深夜独处（不是失眠治疗）

类别二：情绪时刻
━━━━━━━━━━━━━━━━━━━━━━━━
- 蜷缩（不是"缓解焦虑"）
- 迷茫（不是"找到方向"）
- 等待（不是"高效利用时间"）

类别三：过渡时刻
━━━━━━━━━━━━━━━━━━━━━━━━
- 醒来与入睡之间
- 工作与休息之间
- 社交与独处之间
```

### 时刻 vs 功能

```
❌ 功能型音乐：
  "你想专注吗？点这个歌单。"
  → 音乐 = 工具
  → 目的 = 效率

✓ 时刻型音乐：
  "你想待一会儿吗？这里有一个时刻。"
  → 音乐 = 陪伴
  → 目的 = 共在
```

---

## 5.5 产品形态三：Moodify 场景

### 核心概念

```
场景 ≠ 使用场景（学习、健身、睡眠）
场景 = 情绪场景（独处、等待、想念）

Moodify 的场景，是情绪的场景。
```

### 情绪场景的定义

```
场景一：独处
━━━━━━━━━━━━━━━━━━
不是"一个人待着"
不是"需要安静"

是：想和自己待一会儿
是：外面的世界太吵了，想躲进来

音乐特征：
- 包裹感强
- 低频为主
- 留白多


场景二：等待
━━━━━━━━━━━━━━━━━━
不是"等待结果"
不是"等待消息"

是：不知道在等什么，但想等一会儿
是：时间好像停了

音乐特征：
- 循环感强
- 节奏模糊
- 空间感大


场景三：想念
━━━━━━━━━━━━━━━━━━
不是"思念某人"
不是"后悔过去"

是：想起一些东西，但没有具体的人
是：一种淡淡的、说不清的感受

音乐特征：
- 高频有微微的光
- 旋律若隐若现
- 有"远处"的感觉
```

## 5.6 产品形态四：Moodify 演化

### 核心概念

```
演化 = 不是播放列表，而是情绪旅程

播放列表：你选歌，歌单播放
演化：你进入一个情绪起点，
      音乐带你走过一段旅程，
      你不知道终点在哪里，
      但你知道你会"被带过去"。
```

### 演化的运作方式

```
阶段一：进入
━━━━━━━━━━━
用户选择一个情绪起点
（蜷缩 / 迷茫 / 觉醒）
↓
音乐开始播放
从对应的频率出发

阶段二：跟随
━━━━━━━━━━━
音乐根据用户的状态，
自动微调
（变量循环）

阶段三：转折
━━━━━━━━━━━
某个时刻，
一个单音打破循环
↓
这是"破局"点

阶段四：舒展
━━━━━━━━━━━
音场打开
频率分散
用户感到"松了"

阶段五：停留
━━━━━━━━━━━
舒展状态持续
用户可以选择：
- 继续停留
- 回到上一个状态
- 结束
```

### 演化的关键：不需要用户做决定

```
传统播放：用户决定听什么
Moodify演化：音乐引导用户去哪里

用户不需要想：
"我现在该听哪首歌？"
"这首歌适合我吗？"
"我要不要换一首？"

用户只需要想：
"我想被带一会儿"
"我准备好了"
"我想停在这里"

然后音乐会接管。
```

## 5.7 产品的命名体系

Moodify 的产品，用情绪来命名：

```
情绪层级一：蜷缩（Coil）
━━━━━━━━━━━━━━━━━━━━━━━━━━━
- 状态：紧、沉、冷
- 音乐：从最低频开始
- 温度：冷

情绪层级二：迷茫（Lost）
━━━━━━━━━━━━━━━━━━━━━━━━━━━
- 状态：不确定、悬浮
- 音乐：中频为主，频率在移动
- 温度：微温

情绪层级三：觉醒（Awaken）
━━━━━━━━━━━━━━━━━━━━━━━━━━━
- 状态：看见了什么，还不清晰
- 音乐：高频开始出现
- 温度：中性

情绪层级四：舒展（Expand）
━━━━━━━━━━━━━━━━━━━━━━━━━━━
- 状态：打开、松开、呼吸
- 音乐：全频段展开
- 温度：暖
```

### 副标题体系

```
主标题：情绪层级
副标题：更具体的感受

示例：
蜷缩
└── 无力
└── 崩溃边缘
└── 只想躺着
└── 什么都不要

舒展
└── 微光
└── 可以呼吸了
└── 什么都不用做
└── 刚刚好
```

## 5.8 产品的界面哲学

### 界面的三个原则

```
原则一：不催促
━━━━━━━━━━━━━━━━━━
界面不催促用户做任何事

✗ 错误："为你推荐今日精选"
✓ 正确："你今天想待在哪里？"

✗ 错误："根据你的心情，精选歌单"
✓ 正确："蜷缩 / 迷茫 / 觉醒 / 舒展"


原则二：不解释
━━━━━━━━━━━━━━━━━━
不解释音乐是什么

✗ 错误："这首音乐可以缓解焦虑"
✓ 正确：（没有文字，只有封套）

✗ 错误："这首音乐适合睡前听"
✓ 正确：（只有专辑名称）


原则三：不比较
━━━━━━━━━━━━━━━━━━
不给用户"评分"或"数据"

✗ 错误："你今天听了45分钟音乐"
✗ 错误："你今天的心情改善了23%"

✓ 正确：（没有数据，只有音乐）
```
