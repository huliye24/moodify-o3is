# Moodify - 音乐反向解析 Prompt 系统

## 项目概述

将音乐反向解构为 Suno AI 风格的 Prompt，用于音乐风格分析和相似歌曲推荐。

## 核心目标

```
音乐文件 → 乐谱提取 → 比对分析 → Prompt生成 → 相似推荐
```

## 架构设计原则

1. **插件化**: 每个模块可独立替换/升级
2. **分层解耦**: 底层通用，上层专用
3. **接口稳定**: 对外 API 不变，内部实现可换
4. **渐进增强**: 每个阶段都能用，不做"半成品"

---

## 技术栈决策

| 决策项 | 选择 | 理由 |
|--------|------|------|
| 后端语言 | Go + Python | 音频处理用 Python，其他用 Go |
| 数据库 | SQLite | 本地应用最佳，无需部署 |
| 向量库 | Qdrant | 功能全，支持过滤 |
| LLM 集成 | 第二阶段 | 第一阶段快速验证 |
| 多轨分离 | 第二阶段 | 第一阶段先跑通 |

---

## 整体架构

```
┌─────────────────────────────────────────────────────────────────┐
│                         表现层                                  │
│  [Electron]  [REST API]  [WebSocket]                          │
├─────────────────────────────────────────────────────────────────┤
│                        服务层                                  │
│  [AnalysisService]  [SimilarityService]  [PromptService]       │
├─────────────────────────────────────────────────────────────────┤
│                        核心层 (Plugin Host)                    │
│  [Extractor]  [Comparator]  [Encoder]  [Generator]  [VectorStore]│
├─────────────────────────────────────────────────────────────────┤
│                        数据层                                  │
│  [SQLite]  [Qdrant]  [文件系统]                                │
└─────────────────────────────────────────────────────────────────┘
```

---

## 插件类型

| 类型 | 输入 | 输出 | 示例 |
|------|------|------|------|
| Extractor | 音频文件 | 乐谱数据 | BasicPitch, Crepe |
| Comparator | 两个乐谱 | 相似度 | DTW, LCSS |
| Encoder | 乐谱 | 向量 | RuleEncoder |
| Generator | 乐谱 | Prompt | RuleGenerator, LLMGenerator |
| VectorStore | 向量 | 检索结果 | Qdrant, FAISS |
