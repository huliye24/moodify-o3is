# Moodify AI 音乐播放器 - 三年技术路线图

> 视角：2029年，AI 音乐已成为主流
> 目标：构建面向 AI 音乐时代的下一代播放器

---

## 一、愿景定义

**"让 AI 音乐不止于播放，而是完整的感官体验"**

三年后的 AI 音乐具有以下特性：
- 多模态内容：音频 + 动态封面 + AI 歌词 + 情感数据
- 空间音频：杜比全景声、索尼 360RA、 binaural
- 智能元数据：生成模型、Prompt、情感标签、创作参数
- 实时互动：与 Suno/udio 等平台深度集成

---

## 二、核心技术挑战

### 2.1 音频质量优化

| 挑战 | 现状 | 目标 |
|------|------|------|
| AI 音乐多为 AAC/OPUS | 无损支持缺失 | 支持 FLAC/ALAC 源优先 |
| 动态范围过大 | 压缩失真 | 智能动态控制 (Loudness Normalization) |
| 空间音频解码 | 2.0 立体声 | 耳机空间化、扬声器映射 |

### 2.2 元数据与同步

| 挑战 | 现状 | 目标 |
|------|------|------|
| 歌词同步精度低 | 手动时间戳 | AI 自动生成同步时间戳 |
| 跨平台数据孤岛 | 各平台独立 | Suno/udio 本地缓存 + 云同步 |
| AI 音乐信息缺失 | 仅标题/艺术家 | 展示生成 Prompt、模型版本 |

### 2.3 性能与体验

| 挑战 | 现状 | 目标 |
|------|------|------|
| 启动延迟 | 2-3 秒 | < 500ms |
| 内存占用 | 200MB+ | < 100MB |
| 耗电（移动端） | 高 | 降低 50% |

## 三、技术架构设计

### 3.1 分层架构

```
┌─────────────────────────────────────────────────────────────┐
│                    表现层 (UI Layer)                         │
│  React/Vue + WebAudio + Canvas 可视化                       │
├─────────────────────────────────────────────────────────────┤
│                   音频引擎层 (Audio Engine)                   │
│  ┌─────────────┐ ┌─────────────┐ ┌─────────────┐           │
│  │ 解码器抽象   │ │ DSP 处理器  │ │ 输出路由     │           │
│  │ (FFmpeg WASM)│ │ (语音/空间) │ │ (设备选择)  │           │
│  └─────────────┘ └─────────────┘ └─────────────┘           │
├─────────────────────────────────────────────────────────────┤
│                   数据层 (Data Layer)                        │
│  ┌─────────────┐ ┌─────────────┐ ┌─────────────┐           │
│  │ 本地数据库   │ │ 缓存系统    │ │ 同步引擎    │           │
│  │ (SQLite+FS) │ │ (LRU+磁盘) │ │ (CRDT)     │           │
│  └─────────────┘ └─────────────┘ └─────────────┘           │
├─────────────────────────────────────────────────────────────┤
│                   AI 服务层 (AI Layer)                       │
│  ┌─────────────┐ ┌─────────────┐ ┌─────────────┐           │
│  │ 歌词同步    │ │ 音频增强    │ │ 推荐引擎    │           │
│  │ (Whisper)  │ │ (MDX-Net)  │ │ (Embedding) │           │
│  └─────────────┘ └─────────────┘ └─────────────┘           │
└─────────────────────────────────────────────────────────────┘
```

### 3.2 音频处理管线

```
输入源 → 解码 → 音频图处理 → DSP链 → 输出
   │        │         │          │       │
   ▼        ▼         ▼          ▼       ▼
URL/文件  FFmpeg   音频节点    EQ/空间化  扬声器/
          /WAS      (WebAudio)  响度控制   耳机
```

**核心 DSP 节点**：

| 节点 | 功能 | 优先级 |
|------|------|--------|
| LoudnessNormalizer | EBU R128 响度归一化 | P0 |
| Crossfeed | 耳机音场展宽 | P1 |
| ParametricEQ | 7-band 参数均衡器 | P0 |
| ConvolutionReverb | 空间混响 | P2 |
| StereoEnhancer | 立体声增强 | P1 |
| HeadphoneEncoder | 空间音频耳机编码 | P2 |

```typescript
  // 基础信息
  title: string
  artist: string
  
  // AI 生成信息

  modelVersion: string
  prompt: string          // 原始生成 Prompt
  stylePrompt: string    // 风格描述
  generationParams: {
    duration: number
    instrumental: boolean
    temperature?: number

  // 情感数据
  emotion: {
    primary: 'happy' | 'sad' | 'energetic' | 'calm' | 'dark' | 'uplifting'

    tags: string[]        // ['melancholic', 'nostalgic']
  }
  
  // 音频特征
  audioFeatures: {
    key: string          // 'C minor'
    bpm: number
    energy: number       // 0-1
    danceability: number  // 0-1
  }
}
```

## 六、技术债务与风险

### 6.1 主要风险

| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|----------|
| WebAudio 性能瓶颈 | 高 | 中 | WebAssembly 替代关键路径 |
| 版权争议 (AI 音乐) | 中 | 高 | 法律顾问 + 合规框架 |
| 第三方 API 不稳定 | 高 | 高 | 本地缓存 + 离线模式 |
| 移动端 WASM 支持 | 中 | 中 | React Native 回退 |

### 6.2 技术债务

- ~~WebAudio 直接操作~~ → 迁移到音频图抽象
- ~~硬编码 DSP~~ → 插件化架构
- ~~单体数据库~~ → CQRS 分离读/写
