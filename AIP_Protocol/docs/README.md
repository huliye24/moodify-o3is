## 📝 关键决策记录（ Month 1-6 ）

### 决策 1: 为什么选择 ADR 格式？

**背景**：需要标准化决策文档格式

**方案对比**：
| 方案 | 优点 | 缺点 |
|------|------|------|
| 自定义格式 | 灵活 | 不标准 |
| ADR 标准 | 业界标准、工具多 | 需学习成本 |

**决策**：采用 ADR 标准，因为它有成熟工具链和社区支持

**后果**：
- ✅ 决策格式统一
- ✅ 易于工具解析
- ⚠️ 需要教育团队

---

### 决策 2: 为什么用文件系统而不是数据库？

**背景**：AIP 协议状态存储方案选择

**方案对比**：
| 方案 | 优点 | 缺点 |
|------|------|------|
| 文件系统（JSON/YAML） | Git 友好、版本控制、可读 | 查询慢、需自行索引 |
| 数据库（SQLite/PostgreSQL） | 查询快、事务 | Git 不友好、需额外工具 |

**决策**：文件系统 + 索引（Elasticsearch）

**后果**：
- ✅ 完全 Git 友好，历史可追溯
- ✅ 易于备份（直接 git clone）
- ⚠️ 大规模项目需要索引优化
- ⚠️ 并发写需锁机制

---

### 决策 3: 为什么用 Node.js 而不是 Go？

**背景**：工具链开发语言选择

**方案对比**：
| 方案 | 优点 | 缺点 |
|------|------|------|
| Node.js | 跨平台、npm 生态、易写 | 性能一般 |
| Go | 高性能、编译单文件 | 生态小、需编译 |

**决策**：Node.js（工具类应用，性能要求不高，快速开发优先）

**后果**：
- ✅ 开发速度快
- ✅ 跨平台无需编译
- ⚠️ 需要 Node.js 环境
- ⚠️ 大文件处理性能待优化

A: 查看 `2_任务/` 中 `status: "pending"` 的第一个任务

A: 将任务状态改为 `blocked`，在 notes 中记录原因，继续下一个任务

A: 需在任务 notes 中说明原因，并将状态改为 `cancelled`
