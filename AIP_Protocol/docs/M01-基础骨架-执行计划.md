### 目标

### 每日计划

#### Day 1-2: 任务文件分析

**任务**：
- [ ] M01-W3-D1: 分析现有 tasks/ 目录结构
- [ ] M01-W3-D2: 设计任务迁移映射表

**分析内容**：
```
现有 tasks/ 结构：
├── phase1/
│   ├── 01-项目初始化.md
│   ├── 02-核心框架搭建.md
│   └── ...
├── phase2/
├── phase3/
└── logs/

问题：
- Markdown 格式，非 JSON
- 缺少 task_id
- 没有状态字段
- 依赖关系不明确

迁移策略：
1. 将每个 phase 的 .md 拆分为多个原子任务
2. 生成唯一 task_id（T20260413_XXX）
3. 推断状态（已完成 → completed）
4. 提取依赖关系（从前置任务推断）
```

**产出**：
- `tools/migrate-tasks.js` - 迁移脚本
- `tasks/migration-plan.md` - 迁移计划
